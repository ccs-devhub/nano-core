*nano-core* es un core ligero y modular para Discord bots, licenciado bajo la [[file:LICENSE][Mozilla Public License v2.0 (MPL v2)]]. Proporciona la infraestructura básica necesaria para construir bots de Discord totalmente personalizados, con un enfoque en la *extensibilidad*, *adaptabilidad* y la *libertad del usuario*. La *MPL v2* preserva la libertad del software en el core, pero permite que contribuyentes y comunidades desarrollen, compartan e inyecten nuevas funcionalidades bajo cualquier licencia.

#+begin_comment
*Key Features:*
- Minimal, extensible core
- Dynamic command/event handler watching folders for plugins
- Injection of new features (modules) via simple commands or folder additions
- Easy integration of community-driven development
- Designed to be forked, extended, and adapted at will
#+end_comment

* ¿Por qué nano-core? | El problema y nuestra solución
  La mayoría de los bots de Discord actuales son pesados, cargados con innumerables funciones que *nunca* utilizarás, o están bloqueados tras paywalls y suscripciones premium. Para obtener la funcionalidad que necesitas, a menudo terminas añadiendo varios bots, saturando tu Server y dificultando la gestión y el control de su configuración. Las funcionalidades suelen ser monolíticas, difíciles de eliminar o adaptar, y raramente existen comunidades alrededor de funcionalidades individuales.

  nano-core resuelve esto.

  En lugar de descargar un bot sobrecargado innecesariamente, nano-core te anima a desplegar tu propio bot e *inyectar* exactamente las funcionalidades que deseas, /(tal como instalar paquetes en Linux)/. Mediante un sistema sencillo basado en carpetas y comandos, puedes agregar, quitar o reinstalar funcionalidades (modules) en minutos. Cada module puede tener su propia comunidad, documentación y licencia, haciendo que nano-core sea *altamente personalizable*, *modular* y *a future proof*. Construye el bot que necesitas, no más gasto innecesario de dinero, recursos del Server o esfuerzo en funciones que no deseas.

* Licenciamiento y principios comunitarios
  El *core* de nano-core es software libre, protegido por la Mozilla Public License v2.0. Esto garantiza que todo el mundo pueda usar, estudiar, modificar y compartir el código base. Al mismo tiempo, los *modules/extensions* (plugins, commands, events) pueden usar cualquier licencia, reflejando la dedicación de CCS a la *autonomía* y la *participación inclusiva*. Los contribuidores solo deben declarar claramente la licencia de sus modules.

  *Estructura del repositorio:*
  - Archivos core (command/event handlers, injection system): [[file:LICENSE][MPL v2.0]]
  - Modules/extensions: Any license (must specify in folder or header)

* Primeros pasos
  Esta seccion es el arranque rapido; la referencia completa de cada
  comando, clave de configuracion y servicio esta en
  [[file:docs/MANUAL.org][el manual]].

  Requisitos: Node.js >= 20 y una aplicacion de bot creada en el
  [[https://discord.com/developers/applications][Discord Developer Portal]].

  1. Clona e instala:
     #+begin_src shell
     git clone https://github.com/ccs-devhub/nano-core.git
     cd nano-core && npm install
     #+end_src
  2. Secretos (solo por entorno, nunca en JSON):
     #+begin_src shell
     cp .env-template .env
     # DISCORD_TOKEN: portal > Bot > Reset Token
     # CLIENT_ID:     portal > General Information > Application ID
     #+end_src
  3. Configura el bot — abre la ventana de configuracion o edita
     =nano.config.json= directamente:
     #+begin_src shell
     cp nano.config-template.json nano.config.json
     npm run ui    # Dashboard / Config / Modules / Store / Logs
     #+end_src
     En =Config= define el nombre del bot, el =dev_guild_id= (tu server
     de pruebas: los slash commands aparecen al instante), intents,
     base de datos y logging. Colores, paleta y logo del banner y de
     la ventana viven en =nano.style.toml=.
  4. Invita el bot a tu server: portal > OAuth2 > URL Generator con
     los scopes =bot= + =applications.commands= y los permisos que tu
     bot necesite.

* Verificar que todo funciona
  #+begin_src shell
  npm run doctor
  #+end_src
  Revisa cada capa sin conectar el bot: version de Node, config
  valida, secretos presentes, token autenticando contra la API de
  Discord, base de datos, nano-store alcanzable, carga de modules e
  intents (avisa si hay privilegiados por activar en el portal).

  Con todo en verde, arranca y prueba en Discord:
  #+begin_src shell
  npm run dev
  #+end_src
  =/synapse vitals= (salud del core y los modules), =/synapse scan=
  (radiografia del server), =/module list= (estado de los modules).

  Para desarrollo del core: =npm run lint=, =npm run build=,
  =npm run test=.

* Gestionar modules
  #+begin_src shell
  npm run module -- search             # catalogo validado (nano-store)
  npm run module -- install <name>     # instala version exacta revisada
  npm run module -- add ./modules/x    # module local propio
  npm run module -- add <npm-pkg> --allow-external   # NO revisado: bajo tu riesgo
  npm run module -- enable|disable <name>
  npm run module -- outdated           # versiones nuevas validadas
  npm run module -- list
  #+end_src
  Tras activar o desactivar modules, ejecuta =/module sync= en Discord
  para refrescar los slash commands. La guia completa para crear
  modules esta en [[file:docs/MODULE-SPEC.org][MODULE-SPEC]] y las
  leyes del core en [[file:docs/rules/][docs/rules/]].

* Documentacion
  Toda la documentacion es parte del repositorio:
  - [[file:docs/MANUAL.org][MANUAL]] — la referencia completa: cada

  - [[file:docs/MODULE-SPEC.org][MODULE-SPEC]] — el contrato completo

  - [[file:docs/rules/][docs/rules/]] — las leyes de cada subsistema

    modules).
  - [[https://github.com/ccs-devhub/nano-store][nano-store]] — el
    registro curado y su proceso de submission (SUBMISSIONS.md).

* Comunidad y contribuciones
  nano-core es *community-driven*. Cualquiera puede contribuir mejoras
  al core (bajo MPL v2.0) o modules (bajo la licencia que elija). Los
  modules validados se publican en
  [[https://github.com/ccs-devhub/nano-store][nano-store]] via pull
  request. Comparte tus modules, ayuda a otros a aprender y unete a
  las discusiones en [[https://github.com/ccs-devhub][CCS]].

* Invitación final
  No te conformes con bots genéricos y recursos desperdiciados, configura el *Discord bot* más personalizable y modular, adaptado a tus necesidades, con la comunidad CCS apoyando su trayecto. *Tu* defines las características, el licenciamiento y el destino de su bot.
