*nano-core* is a lightweight, modular Discord bot core licensed under the [[file:LICENSE][Mozilla Public License v2.0 (MPL v2)]]. It provides the core infrastructure necessary for building fully customized Discord bots, focusing on *extensibility*, *adaptability*, and *user freedom*. The *MPL v2* preserves software freedom in the core, but allows contributors and communities to develop, share, and inject new features under any license.

#+begin_comment
*Key Features:*
- Minimal, extensible core
- Dynamic command/event handler watching folders for plugins
- Injection of new features (modules) via simple commands or folder additions
- Easy integration of community-driven development
- Designed to be forked, extended, and adapted at will
#+end_comment

* Why nano-core? | The Problem & Our Solution
Most Discord bots today are bulky, packed with countless features you’ll *never* use, or locked behind paywalls and premium subscriptions. To get the functionality you need, you often end up adding several bots, cluttering your server and making it harder to manage and control your setup. Features are monolithic, hard to remove or adapt, and communities around individual functionality rarely exist.

nano-core solves this.

Instead of downloading a bloated bot, nano-core encourages *you* to deploy your own bot and *inject* exactly the features you want, /(just like installing packages in Linux)/. Through an easy folder-based, command-driven system, you can add, remove, or reinstall features (modules) in minutes. Each module can have its own community, documentation, and license, making nano-core *highly customizable*, *modular*, and *future-proof*. You build the bot you need, no more wasting money, server resources, or effort on features you don’t want.

* Licensing & Community Principles
The *core* of nano-core is free software, protected by the Mozilla Public License v2.0. This ensures everyone can use, study, modify, and share the foundational code. At the same time, *modules/extensions* (plugins, commands, events) can use any license, reflecting CCS’s dedication to *autonomy* and *inclusive participation*. Contributors only need to clearly state the license of their modules.

* Getting Started
  This section is the quickstart; the complete reference for every
  command, configuration key, and service is
  [[file:docs/MANUAL.org][the manual]].

  Requirements: Node.js >= 20 and a bot application created in the
  [[https://discord.com/developers/applications][Discord Developer Portal]].

  1. Clone and install:
     #+begin_src shell
     git clone https://github.com/ccs-devhub/nano-core.git
     cd nano-core && npm install
     #+end_src
  2. Secrets (environment only, never JSON):
     #+begin_src shell
     cp .env-template .env
     # DISCORD_TOKEN: portal > Bot > Reset Token
     # CLIENT_ID:     portal > General Information > Application ID
     #+end_src
  3. Configure the bot — open the configuration window or edit
     =nano.config.json= directly:
     #+begin_src shell
     cp nano.config-template.json nano.config.json
     npm run ui    # Dashboard / Config / Modules / Store / Logs
     #+end_src
     In =Config= set the bot name, the =dev_guild_id= (your test
     server: slash commands appear instantly there), intents,
     database, and logging. Colors, palette, and the banner/window
     logo live in =nano.style.toml=.
  4. Invite the bot to your server: portal > OAuth2 > URL Generator
     with the =bot= + =applications.commands= scopes and whatever
     permissions your bot needs.

* Verify everything works
  #+begin_src shell
  npm run doctor
  #+end_src
  Checks every layer without logging the bot in: Node version, valid
  config, secrets present, token authenticating against the Discord
  API, database, nano-store reachability, module loading, and intents
  (warns about privileged ones you must enable in the portal).

  With everything green, run it and test in Discord:
  #+begin_src shell
  npm run dev
  #+end_src
  =/synapse vitals= (core + module health), =/synapse scan= (full
  server snapshot), =/module list= (module states).

  Core development gates: =npm run lint=, =npm run build=,
  =npm run test=.

* Managing modules
  #+begin_src shell
  npm run module -- search             # validated catalog (nano-store)
  npm run module -- install <name>     # exact reviewed version, pinned
  npm run module -- add ./modules/x    # your own local module
  npm run module -- add <npm-pkg> --allow-external   # UNREVIEWED: at your own risk
  npm run module -- enable|disable <name>
  npm run module -- outdated           # newer validated versions
  npm run module -- list
  #+end_src
  After enabling or disabling modules, run =/module sync= in Discord
  to refresh the slash commands. The full module-authoring guide is in
  [[file:docs/MODULE-SPEC.org][MODULE-SPEC]] and the core laws in
  [[file:docs/rules/][docs/rules/]].
